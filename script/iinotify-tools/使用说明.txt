1、安装inotify-tools
	##查看系统版本
		cat /etc/redhat-release
	##配置yum源
		如果系统是centos7
		wget -O /etc/yum.repos.d/epel.repo http://mirrors.aliyun.com/repo/epel-7.repo
		如果系统是centos6
		wget -O /etc/yum.repos.d/epel.repo http://mirrors.aliyun.com/repo/epel-6.repo
	##安装工具
		yum install inotify-tools -y
		
2、修改inotify最大限制
		echo 999999 >/proc/sys/fs/inotify/max_user_watches
3、配置清理脚本
	###将bin文件夹上传至/root目录并添加执行权限
		chmod -R +x /root/bin
	###配置config文件
		vi /root/bin/config
		###配置文件说明
		#监控目录
		file_dir=/test
		#日志目录
		logs_dir=/opt/logs
		#文件监控周期秒
		time_out=120
		#删除的比例必须小于1
		ratio=0.05
		#删除次数临界值必须是正整数
		access=10
4、配置系统定时任务
	vi /etc/crontab
	例如配置每星期1凌晨1点执行加入以下内容
	0 1 * * 1 root cd /root/bin && sh run.sh
	crontab的配置可以使用工具生成https://tool.lu/crontab/
5、重启crond服务
	如果系统是centos7
	systemctl restart crond
	如果系统是centos6
	service crond restart

其他说明：
	定时任务执行周期必须大于文件监控周期，以免后台任务过多卡死
	删除多少的两个参数（删除的比例、删除次数临界值）用来控制删除文件的多少
	删除的比例设置为0时、就取决于删除次数临界值来删除文件
	删除次数临界值为0时、就取决于删除的比例来删除文件
	两者都为0时不删除文件
